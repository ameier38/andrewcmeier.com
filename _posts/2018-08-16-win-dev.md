## Windows Subsystem for Linux
Run bash on Windows.

### Installation
1. Enable Windows Subsystem for Linux.
```
> Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
```
2. Install [Ubuntu](https://www.microsoft.com/en-us/p/ubuntu/9nblggh4msv6?rtc=1)

### References
- [Windows 10 Installation Guide ](https://docs.microsoft.com/en-us/windows/wsl/install-win10)

## Scoop
Command line installer for Windows.

### Installation
1. Set the execution policy.
```
> Set-ExecutionPolicy RemoteSigned -s CurrentUser
```

2. Install scoop.
```
> iex (New-Object net.webclient).downloadstring('https://get.scoop.sh')
```

3. Add extras and versions buckets.
```
> scoop add bucket extras
> scoop add bucket versions
```

### Usage
Search for packages
```
> scoop search vim
```
Install a package
```
> scoop install vim
```

### References
- [Scoop GitHub](https://github.com/lukesampson/scoop)

## Vimium
Google Chrome extension to speed up your Googling. Use Vim keybindings
to navigate around the web.

### Installation
Head to [Vimium homepage](https://vimium.github.io/) and click the 'Install' button.
Or search for 'Vimium' in the [Chrome Web Store](https://chrome.google.com/webstore/category/extensions)
and install from there.

### Usage
- `j,k`: used to scroll up and down respectively.
- `d,u`: used to page up and down respectively.
- `f-{other}`: used to show links on page and then navigate to `{other}` which was the shown link.
- `?`: show the rest of the keybindings and other help.

## Git
Version control for champs.

### Installation
```
> scoop install git
```

### Usage
Initialize a new repository.
```
> mkdir my-repo
> cd my-repo
> git init
```

Add a file to track.
```
> echo "hello" > hello.txt
> git add hello.txt
```

Commit changes.
```
> git commit -m 'added hello.txt'
```

## Kubernetes
Run a local [Kubernetes](https://kubernetes.io/) cluster to test services.

### Installation

1. Install Docker. (See [Docker instructions](#docker-and-docker-compose) for more information)
2. Open the Docker settings.

   ![open-docker-settings](/assets/images/win-dev/open-docker-settings.png){:class="shadow"}

3. Enable Kubernetes.

   ![docker-enable-kubernetes](/assets/images/win-dev/docker-enable-kubernetes.png){:class="shadow"}

### Usage
Use [kubectl](#kubectl) to interact with the local cluster. A local `kubeconfig` file is
created in the `~/.kube` directory when enabling Kubernetes. `kubectl` will use this config
to connect to the local cluster.

## Cmder
Portable console emulator.

### Installation
```
> scoop install cmder
```

### Customization
1. Download [Meslo font](https://github.com/ryanoasis/nerd-fonts/blob/master/patched-fonts/Meslo/M/complete/Meslo%20LG%20M%20Regular%20Nerd%20Font%20Complete%20Windows%20Compatible.otf).

2. Right click on download font and click 'Install'.

   ![install-font](/assets/images/win-dev/install-font.png){:class="shadow"}

3. Open cmder from start menu.

   ![open-cmder](/assets/images/win-dev/open-cmder.png){:class="shadow"}

4. Open cmder settings using keyboard shortcut `Win-Alt-P`.

5. Set the font.

   ![set-cmder-font](/assets/images/win-dev/set-cmder-font.png){:class="shadow"}

6. Set the PowerShell task.

   ![powershell-task](/assets/images/win-dev/powershell-task.png){:class="shadow"}

7. Set the Ubuntu task. Change the path to use your username.

   ![ubuntu-task](/assets/images/win-dev/ubuntu-task.png){:class="shadow"}

8. Pin cmder to the task bar. If it is the first item then you
can access it using the keyboard shortcut `Win-1`.

   ![pin-cmder](/assets/images/win-dev/pin-cmder.png){:class="shadow"}

### References
- [cmder](http://cmder.net/)

## posh-git and oh-my-posh
Better prompt.

### Installation
1. Open PowerShell profile.
```
> vim $PROFILE
```

2. Add posh-git and oh-my-posh.

```powershell
Import-Module oh-my-posh
Set-Theme paradox
```

## Pipenv
Python package management.

### Installation
First make sure Python is installed (see [Python](#python) above).
```
> pip install --user pipenv
```

Check the installation.
```
> pipenv
Usage: pipenv [OPTIONS] COMMAND [ARGS]...

Options:
  --where             Output project home information.
...
```

### Usage
Create a virtual environment.
```
> mkdir my-project
> cd my-project
> pipenv --python 3.7
```

Install a package
```
> pipenv install pandas
```

### References
- [Pipenv homepage](https://docs.pipenv.org/)

## Jupyter Lab
Web based Notebook/REPL.

### Installation
First create a directory, e.g. `notebooks`.
```
> mkdir notebooks
> cd notebooks
```

Use Pipenv to create a virtual environment (see [Pipenv](#pipenv) above.).
```
> pipenv --python 3.7
> pipenv install jupyterlab
```

### Usage
Start Jupyter
```
> pipenv run jupyter lab
```

Install [Jupyter VIM extension](https://github.com/jwkvam/jupyterlab-vim) (I highly recommend).
```
> jupyter labextension install jupyterlab_vim
```

## kubectl
Kubernetes CLI

### Installation
```
> scoop install kubectl
```

### Configuration
1. Add your cluster configuration file to the `~/.kube` directory.
2. Set the `KUBECONFIG` environment variable.
```
> vim $PROFILE
```
```powershell
# "~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1"
$env:KUBECONFIG = "C:\Users\{username}\.kube\{my-config};"
```

### Usage
```
> kubectl get pods 
```

## kubectxwin
CLI for configuring Kubernetes contexts

### Installation
1. Clone the repository.
```
> git clone https://github.com/thomasliddledba/kubectxwin.git
```

2. Add the executable to the PATH.
```
> vim $PROFILE
```
```powershell
# "~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1"
$env:Path += ";C:\{path to kubectxwin repo}\bin"
```

3. Create an alias.
```
> vim $PROFILE
```
```powershell
# "~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1"
New-Alias ktx kubectxwin
```

### Usage
View contexts
```
> ktx ls
```
```
docker-for-desktop
other-context
```

Change context
```
> ktx set docker-for-desktop
```
```
Switched to context "docker-for-desktop".
```

## kubenswin
CLI for configuring Kubernetes namespaces

### Installation
1. Clone the repository.
```
> git clone https://github.com/thomasliddledba/kubenswin.git
```

2. Add the executable to the PATH.
```
> vim $PROFILE
```
```powershell
# "~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1"
$env:Path += ";C:\{path to kubenswin repo}\bin"
```

3. Create an alias.
```
> vim $PROFILE
```
```powershell
# "~\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1"
New-Alias kns kubenswin
```

### Usage
View namespaces
```
> kns ls
```
```
* docker
kube-public
kube-system
openfaas
```

Change namespace
```
> kns set openfaas
```
```
Context "docker-for-desktop" modified.
Active namespace is openfaas in context docker-for-desktop
```
